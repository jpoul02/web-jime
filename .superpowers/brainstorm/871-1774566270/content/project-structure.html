¿Cómo organizamos el nuevo proyecto?

Hay 3 formas de estructurar web-jime + el nuevo formulario de tributos + el backend

A

Monorepo — todo en una carpeta

Una carpeta nueva jime-project/ con subcarpetas: web-jime/ (web existente), tributos-app/ (nuevo formulario), api/ (backend Node.js + PostgreSQL)

A favor

  • Un solo repo, todo junto
  • Fácil compartir tipos y utils
  • Un solo deploy en Railway

En contra

  • Hay que configurar workspaces
B

Dos repos separados

Repo 1: web-jime (queda igual, solo añadimos un link al form)
Repo 2: jime-tributos/ con frontend/ + api/ juntos

A favor

  • web-jime no cambia
  • Deploy independiente
  • Más simple de manejar

En contra

  • Dos repos que mantener
C

Todo dentro de web-jime

El formulario de tributos vive en /tributo dentro del mismo Next.js.
La API es un backend separado (jime-api/) que se conecta a PostgreSQL en Railway.

A favor

  • Una sola web, un solo deploy frontend
  • Navegación integrada entre secciones

En contra

  • web-jime crece bastante
